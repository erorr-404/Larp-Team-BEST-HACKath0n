# Список метрик, які нам потрібно витягнути

## GPS 
 - Lat (щирота)
 - Lng (довгота)
 - Alt (висота)
 - Spd (швидкість)
 - NStats (к-супутників)

## IMU
❗ На дроні їх було кілька, можемо або брати середнє арифметичне або брати дані число з одного. 

- GyrX, GyrY, GyrZ (швидкість обертання навколо осей X, Y та Z. Вимірюється в радіанах за секунду)
- AccX, AccY, AccZ (лінійне прискорення вздовж осей X, Y та Z. Вимірюється в метрах на секунду в квадраті (м/с²), *критично потрібні для інтегрування*)

⚠️ Гравітація в AccZ. Навіть коли дрон просто стоїть на землі й не рухається, AccZ не буде дорівнювати нулю. Він буде показувати приблизно -9.81 м/с² (або +9.81, залежно від орієнтації плати), оскільки акселерометр відчуває земну гравітацію.

## BARO
Це барометр (вимірює тиск). Їх в дроні було два, тому треба або брати середнє арифметичне або брати дані число з одного.

- Alt (висота в метрах, це головне, що нам треба)
- Temp (градуси цельсія, тиск повітря залежить від температури)
- Press (тиск у Паскалях)
- CRt (метри на секунду, Climb Rate, швидкість підйому чи спуску)

💡 *Лайфхак для хакатону*: У вашому завданні є вимога: знайти максимальний набір висоти та максимальну вертикальну швидкість.
Для 3D-візуалізації (вісь Y в Unity) та визначення максимальної висоти обов'язково використовуйте BARO.Alt, а не GPS.Alt. Ваш політ виглядатиме набагато реалістичніше і плавніше.

## BAT
Нам добре було б мати дані з акумулятора, але поки я їх на находив. Ймовірно їх просто немає.
- Volt (вольти, поточна напруга)
- Curr (ампери, поточний струм споживання, залежить від навантаження моторів)
- CurrTot (mAh, загальна к-сть спожитого струму)
